# 📄 Day 6 GUI 창 표시 문제 해결 방법

**작성일**: 2025-11-12  
**문제**: 실행 파일 실행 시 GUI 창이 표시되지 않음  
**상태**: 🔍 원인 분석 중

---

## 🔍 문제 현상

- 실행 파일(`dist/급여명세서생성기`) 실행 시 터미널에 명령어만 생성되고 GUI 창이 열리지 않음
- 로그를 보면 프로그램은 정상적으로 실행되고 있음 (폰트 등록, 대시보드 초기화 등)

---

## 🔍 원인 분석

### 가능한 원인들

1. **macOS 보안 설정 (Gatekeeper)** ⚠️
   - 코드 서명이 없는 실행 파일은 실행이 차단될 수 있음
   - 시스템 환경설정 > 보안 및 개인 정보 보호에서 확인 필요

2. **tkinter 메인 루프 문제** ⚠️
   - `root.mainloop()`가 제대로 실행되지 않을 수 있음
   - macOS에서 tkinter GUI가 백그라운드에서 실행될 수 있음

3. **콘솔 모드 설정** ⚠️
   - `console=False`로 설정되어 있어 오류 메시지가 표시되지 않을 수 있음

---

## ✅ 해결 방법

### 방법 1: 콘솔 모드로 재빌드 (디버깅용)

**build.spec 수정**:
```python
console=True,  # 디버깅을 위해 콘솔 모드로 변경
```

**재빌드 및 실행**:
```bash
pyinstaller build.spec --clean
./dist/급여명세서생성기
```

이렇게 하면 터미널에 오류 메시지가 표시되어 문제를 확인할 수 있습니다.

### 방법 2: macOS 보안 설정 확인

1. **시스템 환경설정** 열기
2. **보안 및 개인 정보 보호** 선택
3. **일반** 탭에서 실행 파일 실행 허용 확인
4. 실행 파일을 우클릭하여 **열기** 선택 (첫 실행 시)

### 방법 3: 직접 실행 확인

터미널에서 직접 실행하여 오류 확인:
```bash
cd /Users/jisungs/Documents/dev/sideprojects/salary_cal
./dist/급여명세서생성기 2>&1 | tee /tmp/payroll_debug.log
```

### 방법 4: Python으로 직접 실행 (개발 환경)

가상환경에서 직접 실행하여 문제 확인:
```bash
source venv/bin/activate
python main.py
```
